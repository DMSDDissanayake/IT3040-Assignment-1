## How to Run

1. Open a terminal in this project folder (`IT23864160`).

2.  Create and activate a virtual environment:

   ```bat
   python -m venv venv
   venv\Scripts\activate
   ```

3. Install dependencies and Playwright browsers:

   ```bat
   pip install -r IT23864160_Requirements.txt
   playwright install
   ```

4. Run the automation. Pass the path to your Excel file and the translator URL:

   ```bat
   python IT23864160-test_automation.py --excel "Assignment 1 - Test cases.xlsx" --url "https://www.pixelssuite.com/chat-translator"
   ```

   If the workbook is elsewhere, use a full path for `--excel`, for example:

   ```bat
   python IT23864160-test_automation.py --excel "C:\Users\...\IT23864160\Assignment 1 - Test cases.xlsx" --url "https://www.pixelssuite.com/chat-translator"
   ```

5. **Default target URL** can also be set with the environment variable `FRONTEND_URL` instead of `--url`.

### Useful options (from the script)

| Option | Purpose |
|--------|---------|
| `--sheet " Test cases"` | Sheet name *(default includes a leading space, matching the template)* |
| `--headless` | Run browser without a visible window |
| `--wait-ms 5000` | Wait after transliteration (ms) |
| `--save-every N` | Save the workbook every N completed rows |
| `--output other.xlsx` | Save results to a different file |
| `--keep-open` | Keep the browser open until you press Ctrl+C *(non-headless)* |

---

## Output

- Results are written back into the Excel workbook (same file as `--excel`, unless you set `--output`).
- Columns updated or added as needed:
  - **Actual output** — text captured from the Sinhala output area
  - **Status** — `PASS` / `FAIL` when an expected value exists; `COLLECTED` when expected is empty; `UI Error` on interaction failures

---

## Test Case Details

- **Focus:** Negative and edge-case style coverage (mixed input, symbols, real-world phrasing, etc.).
- **Comparison:** String equality between actual output and the **Expected** Sinhala column — spacing, spelling, or formatting differences count as **FAIL**.

### Typical scenarios covered

- Mixed language inputs (Singlish + English)
- Spelling variations
- Emojis and symbols
- Domain-style phrases (e.g. banking, travel, apps)
- System-style text (errors, logs)
- Numeric and date-like inputs

---

## Important Notes

- The tool uses **strict** comparison: small differences in Sinhala output produce **FAIL**.
- Some **FAIL** results are expected when the live site’s transliteration differs from your documented expected text.
- Flaky runs can happen due to **network**, **page load**, or **timing** — increase `--wait-ms` / `--retries` or run without `--headless` to debug.
